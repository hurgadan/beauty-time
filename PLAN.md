# Beauty-Time MVP Plan

## 1. Цель продукта
Сделать MVP SaaS для салонов и частных мастеров (DE/EU): CRM + онлайн-запись по ссылке + email-напоминания и подтверждение визита.

## 2. Границы MVP (что входит в первую версию)
1. CRM для салона/мастера:
- услуги, цены, длительность, буферы;
- мастера и их рабочие часы;
- календарь и управление записями;
- карточка клиента и история посещений.
2. Клиентский booking:
- запись по ссылке;
- выбор услуги, мастера (опционально), даты и времени;
- подтверждение записи через OTP/magic link.
3. Уведомления:
- email после создания записи;
- напоминания `T-24h` и `T-2h`;
- подтверждение визита из письма.
4. Multi-tenant и безопасность:
- изоляция данных между салонами;
- tenant-контекст только из JWT.

### 14. Этап 14. Качество, тестирование и приемка MVP
1. [ ] Unit-тесты:
- цель: быстро проверять доменную логику без поднятия БД и HTTP;
- проверить алгоритм слотов (рабочие часы, длительность, буферы, пересечения, границы дня);
- проверить timezone-логику (`Europe/Berlin`, переходы DST);
- проверить OTP-логику (TTL, неверный/просроченный OTP, лимиты).
2. [ ] Integration-тесты:
- цель: проверить взаимодействие модулей приложения с реальной БД и инфраструктурой;
- проверить полный backend booking flow;
- проверить tenant isolation;
- проверить соответствие API-контрактам.
3. [ ] E2E:
- цель: проверить ключевые пользовательские сценарии через публичные API;
- CRM-сценарии (login, услуги, мастера, расписание, управление записью);
- booking-сценарии (ссылка, выбор, запись, OTP/magic link подтверждение, отражение в CRM).
4. [ ] Зафиксировать и внедрить тестовую инфраструктуру e2e:
- e2e в Docker-окружении с Postgres 18 + `tmpfs`;
- данные создаются в тестах (`beforeEach`/фабрики), без глобальных seed;
- полная очистка БД в `beforeEach`;
- автоматический прогон миграций перед e2e;
- полная изоляция тестов и фабрики для повторяющихся сценариев.
5. [ ] Приемочные критерии MVP:
- запись клиента <= 2 минут;
- отсутствуют двойные бронирования;
- email-напоминания стабильно отправляются с retry.
